Kotlin 島深度之旅 30 天

在去年水深火熱的 9 月，鐵人賽開賽了，說真的那個月真的生不如死啊…
因為我其實沒屯什麼文章… 要挑戰的主題又是全新的 Kotlin 的這個語言，其實一開始是因為 Kotlin 讀書會的團長邀請才參加的，但其實一開始也是很猶豫，因為怕很累，沒時間，Kotlin 在目前的後端上也無法取代 Java，畢竟 Kotlin 還是被 Google 認證在 Adroid 開發上是第一選擇，不過最後還是決定參加了，畢竟說真的我覺得 Java 在台灣還真的比較沒有舞台了…可能大家都是一群很資深的人才寫 Java 了 xd…去參加 Java Conf 可以很明確的感受到，一票人還處在 Java 8 “之前” 的世界，這真的是好處也是壞處呢…XD

anyway 在寫作的過程中，還真的天天都覺得沒啥人在看我的文…我何必這麼辛苦呢

在這鐵人賽的過程中…我至少看了 2x 個影片, 至少有 30 小時以上吧…包含一個完整 4 小時的 tutorial，和很多 Spring One with Kotlin 的影片和 Josh Long 每個 Bootiful Kotlin

還看了很多本書…除了讀書會用的那本(Kotlin Programming: The Big Nerd Ranch Guide), 還看了 Kotlin in Action(部分比較難解釋的…例如泛型, receiving by lambda 等等), Kotlin Cookbook, Kotlin 從入門到項目實戰 等等…大概有兩本是 80% 看完了…

為什麼要看這麼多東西…? 因為我真的很怕誤人子弟…

但也深刻的體會到，看完書跟你要自己寫一套出來解釋，真的是完全不懂的事，如今我也可以大聲的說，我完成啦！！

也很幸運的，作品得到評審的肯定，得到了佳作，可能真的參加的越痛苦，越能得獎吧 orz…拜偷好歹我也很用心…只是可能 Kotlin 太冷門了 XD，沒啥人要看

因為得獎的都可以出書，但出書可能是 3 倍鐵人賽的痛苦程度…我覺得我再看看吧 XD…
